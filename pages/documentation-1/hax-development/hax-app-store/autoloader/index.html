Autoloader is waaayyy easier to implement then the other parts of the appstore spec. All it is is an array or object of references to elements that HAX should forcibly load the definitions of once it's done bootstrapping. Here's a basic example with the steps as to what HAX does with this information

  1. HAX loads the appstore definition
  2. HAX finishes its bootstrap and is ready to use
  3. It then loops through each autoloader element and does the following:
    • attempts to dynamically import the definition of the element
    • places a copy of the tag it in the slot of a tag called `hax-autoloader `
    • waits till it reads the definition from the element via standard events (see: HAXWiring.js )
    • Deletes the element from the DOM
